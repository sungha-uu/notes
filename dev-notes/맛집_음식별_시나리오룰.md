# 음식별 맛집 시나리오룰.md
> 채널: 맛집 유튜브 채널 | 유형 코드: `RESTAURANT_FOOD` | 시나리오 파일: B (음식별)

---

## 1. 콘텐츠 목적
"삼겹살 먹고 싶은데 어디가 제일 맛있지?", "진짜 맛있는 냉면집 알고 싶다"처럼
특정 음식을 기준으로 전국 또는 지역 내 최고 맛집을 큐레이션한다.

---

## 2. 음식 분류 체계

### 한식 메인
| 카테고리 | 세부 음식 |
|----------|----------|
| 국물 요리 | 설렁탕, 곰탕, 순대국밥, 갈비탕, 해장국, 감자탕, 추어탕 |
| 구이류 | 삼겹살, 갈비, 한우, 오리구이, 장어구이 |
| 면류 | 냉면, 막국수, 칼국수, 짜장면, 짬뽕, 수제비 |
| 해산물 | 회, 게장, 대게, 굴, 조개구이, 갈치조림 |
| 밥류 | 비빔밥, 돌솥밥, 솥밥, 김치찌개 백반 |
| 분식 | 떡볶이, 순대, 튀김, 김밥 (전문점 기준) |

### 특식/트렌드
| 카테고리 | 세부 음식 |
|----------|----------|
| 퓨전/트렌드 | 마라탕, 마라샹궈, 탄탄면, 흑당 디저트 |
| 디저트 | 카페 케이크, 빙수, 마카롱, 크로플 |
| 간식 | 핫도그, 붕어빵, 호떡, 길거리 음식 |

### 지역 특산 음식 (음식별 시나리오에서 함께 다룸)
- 전주 비빔밥, 춘천 닭갈비, 안동 찜닭, 속초 아바이순대, 부산 밀면 등

## 3. 수집 소스 (우선순위 순)

### 1순위 — 전문 음식 리뷰 플랫폼
- 망고플레이트 (음식 카테고리 특화 필터 활용)
- 식신 (음식별 랭킹 상위 50 활용)
- 네이버 지도 (음식 카테고리 검색 + 별점 4.0 이상)
- 다이닝코드 (빅데이터 맛집 분석)

### 2순위 — 음식 전문 미디어
- 백종원 유튜브 / 골목식당 방영 식당 (방영 후 6개월 이내)
- 수요미식회, 맛있는 녀석들 등 TV 맛집 프로그램 출연 식당
- 식품 전문 매거진 (맛있는 오 등)
- 미쉐린 가이드 한국판 선정 식당

### 3순위 — 보조 참고
- 인스타그램 음식 전문 계정 (#음식명맛집) — 팔로워 5만 이상 계정 우선
- 유명 푸드 블로거 포스팅 — 광고 여부 명시 확인 필수

---

## 4. 수집 필터 규칙

### 포함 기준
- 해당 음식 전문점 또는 해당 음식이 대표 메뉴인 식당
- 별점 4.0 이상 + 리뷰 100개 이상 (전국구 기준)
- 해당 음식 맛에 대한 구체적 리뷰 다수 존재 (단순 "맛있어요" 아닌 묘사 리뷰)
- 해당 음식 특유의 조리법·재료가 차별화된 곳 우선

### 제외 기준
- 음식이 사이드 메뉴이거나 비전문점
- 오픈 3개월 미만 (리뷰 누적 부족)
- 최근 음식 품질 저하 리뷰 급증 식당
- 단일 음식만 취급하지 않는 뷔페·코스 레스토랑 (별도 편성)

---

## 5. 영상 1편 구성 규칙

| 항목 | 규정 |
|------|------|
| 영상 길이 | 2분 ~ 3분 |
| 소개 맛집 수 | 1편당 전국 5곳 OR 서울 집중 7곳 |
| 구성 흐름 | 훅-음식 클로즈업(10초) → 음식 소개(15초) → 맛집 리스트(90~120초) → 총정리(20초) → CTA(10초) |
| 제목 형식 | `진짜 {음식명} 맛집 N곳 \| {지역} 현지인 추천` 또는 `전국 {음식명} 맛집 순위` |
| 썸네일 키워드 | 음식 클로즈업 사진 + "맛집 N곳" 텍스트 |

---

## 6. 맛집 1곳당 수집 정보 항목

```
- 식당명 (한글 + 영문)
- 해당 음식 대표 메뉴 + 가격
- 이 집만의 차별점 (육수, 재료, 조리법 등 구체적으로)
- 주소 + 지역 (네이버 지도 링크)
- 영업시간 + 정기휴무
- 대기시간 여부 (웨이팅 필요 여부)
- 주차 가능 여부
- 추천 이유 한 줄 (예: "48시간 끓인 사골 육수로 유명한 설렁탕 맛집")
```

## 7. 스크립트 작성 규칙

- 어투: 음식 전문 유튜버 느낌, 맛 표현이 생생하고 구체적이어야 함
- 음식 묘사 시 감각적 언어 사용 (육즙이 터지는, 쫄깃한, 깊고 진한 등)
- 각 식당마다 "이 집만의 특별한 점" 반드시 언급
- 음식별 편당 "이런 분께 추천" 타겟 멘트 포함 (예: "해장이 필요할 때", "데이트 코스로")
- 금지 표현: 특정 식당을 "최고", "1등" 등 서열화 단정 표현
- 필수 포함: "가격·메뉴는 변동될 수 있으니 방문 전 확인하세요"

## 8. 영상 프롬프트 생성 규칙 (영상 에이전트 전달용)

```
유형: 맛집 (음식별)
음식 카테고리: {음식명}
소개 식당: {식당1}, {식당2}, ..., {식당N}
배경 스타일: 음식 중심, 클로즈업 많이 활용, 식욕 자극 조명 (따뜻한 황금빛)
영상 요소: 음식 클로즈업 → 내부 인테리어 → 사람들 먹는 장면 → 주소 자막
텍스트 스타일: 굵은 고딕, 음식명은 빨간색 또는 오렌지 강조, 식당명은 흰색 박스
나레이션 톤: 먹방 느낌의 생생하고 식욕 자극하는 목소리, 약간 흥분된 톤
배경음악: 경쾌하고 식욕 돋우는 재즈 or 팝 (Mubert: food cooking 태그)
전환 효과: 빠른 컷, 음식 클로즈업 줌인 필수
말미 구성: 소개된 음식·식당 빠르게 리캡 + 자막 정리
```

## 9. 지역별 vs 음식별 중복 처리 규칙

동일 식당이 두 시나리오에 모두 해당할 경우:
- **같은 달에 동일 식당 중복 등장 금지**
- 지역별 편에 등장했다면 최소 2개월 후 음식별 편에 등장 가능
- 메인 에이전트가 `중복_식당_로그.json`으로 관리

## 10. 품질 체크리스트 (메인 에이전트 검수용)

- [ ] 해당 음식이 식당의 대표 메뉴인지 확인
- [ ] 맛 차별점이 구체적으로 서술되었는가
- [ ] 모든 식당 현재 영업 중인지 확인
- [ ] 음식 클로즈업 영상이 충분히 포함되었는가
- [ ] 지역별 시나리오와 2개월 내 중복 식당 없는지 확인
- [ ] 영상 길이 2~3분 범위인가
